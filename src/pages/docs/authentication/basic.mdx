# Basic authentication settings

This topic describes some basic authentication-related configurations.

## Email suffix

### How to configure

<Steps>

<div>
<Nav name="system-settings" />
</div>

<div>
Navigate to **System settings > Authentication > Basic**.
</div>

<div>
In the **Email suffix** field, type the email suffix, such as "example.com".
<img src="/images/docs/admin-guides/system-settings/auth/basic-email-suffix.png" alt="image" width="450"/>
</div>

<div>
Click **Submit**.
</div>

</Steps>

### Usage scenarios

If the user from the third-party authentication does not have an email field configured, the email suffix is used to automatically generate the user email address by combining the username with the suffix.


## Forgot password URL

### How to configure

<Steps>

<div>
<Nav name="system-settings" />
</div>

<div>
Navigate to **System settings > Authentication > Basic**.
</div>

<div>
In the **Forgot password URL** field, type the URL for the forgot password link, such as "https://&#8203;forgot-password.example.com".
<img src="/images/docs/admin-guides/system-settings/auth/basic-forgot-password.png" alt="image" width="450"/>
</div>


<div>
Click **Submit**.
</div>

</Steps>

### Usage scenarios

When the user clicks the _[Forgot password]()_ on the login page, they will be redirected to this URL.
<img src="/images/docs/admin-guides/system-settings/auth/login-page-forgot-password.png" alt="image" width="350"/>



## Login redirection
### How to configure

<Steps>

<div>
<Nav name="system-settings" />
</div>

<div>
Navigate to **System settings > Authentication > Basic**.
</div>


<div>
In the **Login redirection** field, check to enable user login redirection.
<img src="/images/docs/admin-guides/system-settings/auth/basic-login-redirection.png" alt="image" width="450"/>
</div>

<div>
Click **Submit**.
</div>

</Steps>

### Usage scenarios
When an unauthenticated user accesses JumpServer, if checked, they will be prompted to either **Cancel** and return to the default login page or **Confirm** to proceed with third-party authentication. If unchecked, they will be directly redirected to the third-party authentication page.
<img src="/images/docs/admin-guides/system-settings/auth/redirection-page.png" alt="image" width="450"/>
