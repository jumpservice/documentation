# About assets

In JumpServer, **Assets** are the fundamental units of management and control, representing various IT resources that users need to access through JumpServer. **Assets** can be physical devices or virtual resources, such as hosts, network devices, databases, cloud services, and websites.

JumpServer supports the following asset types:
* **Host**: A physical server or virtual machine, such as Linux, Unix, or Windows.
* **Device**: A network device, such as a router, switch (e.g., Cisco, H3C, Huawei).
* **Database**: A database server, such as MySQL, MariaDB, PostgreSQL, Oracle, SQL Server, Redis, MongoDB, DB2, Dameng, or ClickHouse.
* **Cloud**: A cloud service, such as Kubernetes or VMware vSphere.
* **Web**: A website or web application.
