# Push account

This topic describes push account configurations.

## Prerequisites
At least one account must be added to the asset before executing push account tasks. Automation tasks will prioritize accounts marked as privileged for execution.
Recomanded account privilege requirement are as follows:

<HorizontalTable 
headers={['Asset Type', 'Recomanded account privilege requirement', 'Protocol requirement']} 
rows={[
    ["Linux", "Root or sudoers", 'SSH enabled'], 
    ["Windows", "Account in Administrator group", 'SSH or WinRM enabled'], 
    ["Unix", "Root or user with ALL PRIVILEGES", 'SSH enabled'], 
    ["Mongodb", "'userAdmin' role", ''], 
    ["Mysql/MariaDB", "Root or user with ALL PRIVILEGES'", ''], 
    ["Postgresql", "Postgres or other superuser", ''], 
    ["Oracle", "SYSDBA", ''],
    ["SQLServer", "Membership in 'sysadmin'", '']
]}
/>

For **Minimum account privilege requirement** of each automation template. See [Platform settings](/docs/platform-settings/automation-settings).

## Create a push account task

<Steps>

<div>
<Nav name="pam" />
</div>

<div>
In the left menu, click **<Icon name="search" /> Push accounts**.
</div>

<div>
On the right page, click the **Account push tasks** tab.
</div>

<div>
Above the table, click **Create**.
</div>

<div>
Type a **Name** for the task in the popup.
</div>

<div>
Optionally, choose **Asset nodes** to push accounts for. All assets under the selected nodes will be included.
</div>

<div>
Optionally, choose **Assets** to push accounts for.
</div>

<div>
Type the **Account** names you want to push.
</div>

<div>
Choose a method to set the secret: whether to use the existing secret or generate a new one.
</div>

<div>
Choose **Secret Type**. The secret type can be either **Password** or **SSH Key**.
</div>

<div>
If using Password, type the **Password** in the field or set **Random generate rules**.
If using SSH Key, type the **SSH Key** in the field.
</div>

<div>
If the target assets are host-type assets, you can configure additional push parameters for the task.

<HorizontalTable 
headers={['Asset type', 'Settings']} 
rows={[
    ["Windows", "User's group."], 
    ["Linux/MacOS/Unix/AIX/BSD", "Shell, Sudo permissions, Home directory, User Group, User ID"], 
]}
/>
</div>

<div>
Optionally, disable or enable **Periodic**, where you can set the **Crontab** or **Interval** to schedule the task. 
If both interval and crontab are set, crontab is prioritized.
</div>

<div>
Optionally, disable or enable **Active**, which determines whether the task is active or not.
</div>

<div>
Click **Submit** or **Save & Continue**.
</div>

</Steps>
