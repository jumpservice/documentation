# Command storage

JumpServer allows administrators to configure command storage for saving command logs generated during asset sessions.

The following command storage types are currently supported:
- Built-in types:
  - Local (stores command logs locally)
  - Null (does not store command logs)
- Cloud or external storage types:
  - Elasticsearch

## Create a command storage
<Steps>
<div>
<Nav name="system-settings" />
</div>

<div>
In the left menu, click **<Icon name="Storage" /> Storage**.
</div>

<div>
On the right page, click **Command storage** tab.
</div>

<div>
Click **+ Create**, and select the desired command storage type.
</div>

<div>
Fill in the required information for the selected command storage type.
</div>

<div>
Click **Submit**.
</div>
</Steps>

## Set a default command storage
After setting a default storage, newly registered components will automatically use it. The command storage type of existing components will remain unchanged.

<Steps>
<div>
<Nav name="system-settings" />
</div>

<div>
In the left menu, click **<Icon name="Storage" /> Storage**.
</div>

<div>
On the right page, click **Command storage** tab.
</div>

<div>
Find the command storage you want to set as default.
</div>

<div>
In the Actions column, click the **···** icon, then click **Set as default**.
</div>

</Steps>
