# User Login

JumpServer supports setting user login rules based on the user's login IP range and login time range when logging into JumpServer. User login rules can be set for all users, specific users, and users matching certain attributes. These rules can be set on the user details page or in the permissions management module. When a JumpServer user logs in and matches a user login rule, the specified action of that rule will be executed. If no corresponding rule is matched, the user will proceed with normal execution. User login rules are effective when set at the global organization level.

![user-login](/images/docs/acls/user-login/user-login.png)

Detailed Parameter Explanation:
| Parameter    | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Name     | Specify the name of this login rule.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Priority | Specify the priority level at which this login rule takes effect, with values ranging from 1 to 100. Smaller numbers indicate higher priority.                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| User     | Specify the users that this login rule should match. **All Users**: This login rule matches all users. **Specific Users**: This login rule matches specific users. **Attribute Filtering**: This login rule matches users based on attribute rules.                                                                                                                                                                                                                                                                                                                                          |
| IP Setting | Specify the restricted login IP ranges for this login rule, formatted as a comma-separated string. "*" indicates a match for all IPs. Example: `192.168.10.1, 192.168.1.0/24, 10.1.1.1-10.1.1.20, 2001:db8:2de::e13, 2001:db8:1a:1110::/64`. This IP refers to the IP the user logs in with.                                                                                                                                                                                                                                  |
| Time period | Specify the restricted user login period for this login rule.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| Action   | Specify the action to be taken when this login rule is executed: **Reject**: Deny the user login when matched by the above rule. **Accept**: Allow the user to log in when matched by the above rule. **Review**: Send a work order to the approver when the user login matches the above rule. Allow the user to log in after approval. **Notify**: Send a notification to the specified user when the user login matches the above rule.                                                                                  |
| Active   | Specify whether this login rule is enabled or not.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
