# Installation guide for JumpServer

This topic describes how to install JumpServer, including both the open-source and enterprise editions.

## Install with a one-line command

This section introduces how to install the open-source version of JumpServer using a one-line command.

<Steps>

<div>
Prepare a clean Linux server ( 64bit, >= 4c8g ).
</div>

<div>
Log in to the Linux server using the "root" or another user with superuser privileges.
</div>

<div>
Change to the `/opt` directory.
</div>

<div>
Run the command to install the latest open-source version of JumpServer. Please wait a moment ☕️…
```sh
curl -sSL https://github.com/jumpserver/jumpserver/releases/latest/download/quick_start.sh | bash
```
</div>

<div>
Please open your Chrome browser, enter "http://\<your-server-ip\>" in the address bar.
<Alert type="note">
Please replace `<your-server-ip>` with your actual server IP address.
</Alert>
</div>

<div>
Log in as an Administrator using the following default credentials.
* username: `admin`
* password: `ChangeMe`
</div>

<div>
For security, you will be prompted to change your password after logging in. Once you've changed it and logged in again, you can start your JumpServer journey.
</div>

<div>
At this point, you have successfully installed JumpServer. 
</div>

</Steps>

## Install with an offline package
This section describes how to install the Enterprise Edition of JumpServer using an offline package. 

<Steps>

<div>
Prepare a clean Linux server ( 64bit, >= 4c8g ).
</div>

<div>
Log in to the Linux server using the "root" or another user with superuser privileges.
</div>

<div>
Change to the `/opt` directory.
</div>

<div>
Download the JumpServer Enterprise offline installation package.

<Tabs
  titlesMap={{
    web: "Web",
    wget: "wget",
  }}
>
  <Tab titleKey="web">
    <Steps stepLevel="2">
      <div>
      Log in to https://download-pkgs.jumpserver.com using the username and password provided by the support team.
      </div>

      <div>
      Click filename to download the offline installation package for the required version.
      </div>

      <div>
      Upload the offline installation package to the `/opt` directory on the deployment server.
      </div>
    </Steps>
  </Tab>

  <Tab titleKey="wget">
    <Steps stepLevel="2">
      <div>
      Log in to https://download-pkgs.jumpserver.com using the username and password provided by the support team.
      </div>

      <div>
      Right-click filename and select "Copy link address" to copy the download link for the required version of the offline installation package.
      </div>

      <div>
      Run the following command to download the offline installation package.
      ```sh
      cd /opt && wget --user={username} --password={password} -O jumpserver-ee-v4.8.1-x86_64.tar.gz "https://download-pkgs.jumpserver.com/download?file=jumpserver-ee-v4.8.1-x86_64.tar.gz"
      ```
      <Alert type="note">
      Please replace `{username}` and `{password}` with the actual username and password provided by the support team.
      </Alert>
      </div>
    </Steps>
  </Tab>
</Tabs>

</div>

<div>
Extract the offline installation package.
```sh 
tar -zxvf jumpserver-ee-v4.8.1-x86_64.tar.gz
```
</div>

<div>
Change to the extracted directory.
```sh
cd jumpserver-ee-v4.8.1-x86_64
```
</div>

<div>
Perform the installation, the installation process can be completed using all default options. After the installation is complete, you can adjust the settings by modifying the configuration files. Now, you may need to wait a moment ☕️ ...
```sh
./jmsctl.sh install
```
</div>

<div>
At this point, running `docker ps` will show the following three containers running:
* jms_core
* jms_postgresql
* jms_redis
</div>

<div>
Start all services.
```sh
./jmsctl.sh start
```
</div>

<div>
Please open your Chrome browser, enter "http://\<your-server-ip\>" in the address bar.
<Alert type="note">
Please replace `<your-server-ip>` with your actual server IP address.
</Alert>
</div>

<div>
Log in as an Administrator using the following default credentials.
* username: `admin`
* password: `ChangeMe`
</div>

<div>
For security, you will be prompted to change your password after logging in. Once you've changed it and logged in again, you can start your JumpServer journey.
</div>

<div>
At this point, you have successfully installed JumpServer. 
</div>
</Steps>
