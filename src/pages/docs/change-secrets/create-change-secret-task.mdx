# Create account secret change task <Icon name="xpack-lg" />

**Change secrets** refers to bulk modification of account login secrets and keys on hosts. Enables centralized management across multiple asset accounts, with custom rotation cycles and optional email notifications for task results.

<Steps>

<div>
<Nav name="pam" />
</div>

<div>
In the left menu, click **<Icon name="ChangePassword" /> Change secrets**.
</div>

<div>
On the right page, click the **Change secret tasks** tab.
</div>

<div>
Click **+ Create**.
</div>

<div>
In the **Name** field, type the task name.
</div>

<div> 
In the **Accounts** field, type one or more usernames for which you want to change the secrets.
</div>

<div>
In the **Assets** field, select one or more assets. The task changes secrets for all accounts under the selected assets.
</div>

<div>
In the **Nodes** field, select one or more nodes. The task changes secrets for all accounts under the selected nodes and their subnodes.
</div>

<div>
In the **Secret strategy** field, you can choose "Specific secret" to set it manually, or "Random generate" to generate it randomly.
</div>

<div>
In the **Secret type** field, you can choose either "Password" or "SSH key".
</div>

<div>
In the **Password** field, type a password. (Specific secret & Password) 
</div>


<div>
In the **SSH key strategy** field, you can choose "Replace (Replace only keys pushed by JumpServer)" or "Empty and append SSH KEY". (SSH Key)
</div>

<div>
In the **Private key** field, paste your private key. (Specific secret & SSH key)
</div>

<div>
In the **Key password** field, type the private key password. (Specific secret & SSH key)
</div>

<div>
In the **Password rules** field, set password rules, including length, uppercase and lowercase letters, numbers, special characters, and excluded characters. (Random generate & Password) 
</div>

<div>
In the **Parameters** field, configure secret change parameters.
<Alert type="note">
Parameters can be configured only when selecting a Host-type asset or a node containing Host-type assets.
</Alert>
</div>

<div>
In the **Periodic** field, check to enable scheduled execution.
</div>

<div>
In the **Check connection after change** field, check to verify the connection after the push task is completed.
</div>

<div>
In the **Active** field, check to enable the task.
</div>

<div>
In the **Description** field, type the task description.
</div>
    
<div>
Click **Submit**.
</div>

</Steps>
